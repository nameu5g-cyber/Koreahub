// turbo-all

Этот воркфлоу автоматизирует проверку данных и "подъем" тестовой среды KoreaHub.

### Шаги:
1. **Проверка окружения**:
   - Убедиться, что файл `.env.local` существует и содержит `GEMINI_API_KEY`.
2. **Комплексный Аудит Соотвествия (Full Compliance Audit)**:
   - **Валидация JSON**: Проверить файлы `attractions.json`, `food.json`, `telecom.json`.
   - **Структурный аудит**: Убедиться в наличии полей `id`, `verticalCode`, `category`, `region`.
   - **Контроль переводов**: Проверить полноту полей `ru`, `en`, `ko` в блоках `name` и `description`.
   - **ОТЧЕТ О НЕСООТВЕТСТВИИ**: В случае обнаружения ошибок, агент обязан:
     - Провести **построчный скан (line-by-line scan)** файла для поиска точного места ошибки.
     - Вывести отчет, включающий: 1) Тип ошибки, 2) ID записи, 3) Номер строки, 4) Причину несоответствия.
3. **Синхронизация (Full Data Sync)**:
   - Подтвердить, что все изменения из планов интегрированы в файлы данных.
4. **Проверка зависимостей**:
   - Убедиться, что папка `node_modules` существует. Если нет — вызвать `npm install`.
5. **Запуск сервера**:
   - Проверить порт 3000 (`netstat -ano | findstr :3000`).
   - Если свободен, запустить: `cmd /c "npm run dev"`.
   - Если занят, вывести: "🚀 Сервер уже запущен на http://localhost:3000".

### Пример использования:
Просто напишите `/up` в чате, когда нужно проверить готовность проекта к деплою.
